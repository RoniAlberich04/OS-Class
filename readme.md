# Zoo Concurrency Simulation

A Python program demonstrating concurrency concepts including starvation and deadlock through a zoo visitor management system.

## Overview

This simulation models a zoo with:
- **3 gates** (threads) processing visitors concurrently
- **2 attractions**: Open Area (300 capacity) and Theater (30 capacity)
- **4 visitor types**: Gold, Silver, School, Regular (priority-based scheduling)

## Features

- Multi-threaded gate processing
- Priority queues for visitor management
- Resource contention simulation
- Starvation detection
- Real-time logging
